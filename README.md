# nikahin

> A Custom Super Premium E-Wedding Invitation Powered by AI.

A PHP 8 / MySQL e-wedding invitation application with Claude-powered design generation. Couples can register, fill in their profile, and receive a fully personalized digital invitation — either from a curated template (Basic) or a unique design generated by Claude AI.

## Features

**For couples:**
- Passwordless OTP authentication (email or phone)
- Two pricing tiers: **Basic Template (IDR 50K)** and **AI-Generated (IDR 150K)**
- 7-step guided form with auto-save
- Photo uploads (groom, bride, prewedding, reference)
- Live preview with copy editing
- Public invitation URL (`/v/{slug}`)
- Personalized guest links (`/v/{slug}/g/{token}`)
- RSVP tracking + guestbook moderation
- Digital gift account display (BCA, GoPay, OVO, etc.)

**For guests:**
- Mobile-first responsive design
- RSVP form with attendee count and event choice
- Guestbook with 280-character messages
- Copy-to-clipboard for gift account numbers

**For admins:**
- Dashboard overview (users, invitations, revenue, generation queue)
- Invitation review and flagging
- User suspension/restoration
- AI generation logs (tokens used, model version, errors)
- Guestbook moderation
- Audit log

## Tech Stack

- **Backend:** PHP 8.1+ (no framework, vanilla PHP)
- **Database:** MySQL 5.7+ / MariaDB 10.3+
- **Frontend:** jQuery 3.7, vanilla CSS (Tactile Maximalism design system)
- **AI:** Claude (Anthropic) via REST API — `claude-sonnet-4-6` (default; swap to `claude-opus-4-7` for premium quality)
- **Auth:** OTP (one-time password) sent via SMS or email
- **Server:** Apache (with mod_rewrite, mod_headers) — Nginx config in [Going to Production](#going-to-production)

## Requirements

- PHP **8.1+** with extensions: `pdo`, `pdo_mysql`, `gd` or `imagick`, `curl`, `json`, `mbstring`
- MySQL **5.7+** or MariaDB **10.3+**
- Apache with `mod_rewrite` and `mod_headers` enabled (or equivalent Nginx setup)
- Writable `uploads/` directory (chmod 755 on the dir, 644 on files)
- (For AI tier) Anthropic API key — get one at https://console.anthropic.com

## Installation

### 1. Clone or extract files

```bash
# Place all files in your web server document root, e.g.
/var/www/html/nikahin/
```

### 2. Create the database

```bash
mysql -u root -p
```

```sql
CREATE DATABASE nikahin CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'nikahin'@'localhost' IDENTIFIED BY 'yourpassword';
GRANT ALL PRIVILEGES ON nikahin.* TO 'nikahin'@'localhost';
FLUSH PRIVILEGES;
exit;
```

### 3. Import the schema

```bash
mysql -u nikahin -p nikahin < database/schema.sql
```

This creates all tables and seeds an admin account at `admin@nikahin.local`.

### 4. Configure the app

Edit `config/config.php`:

```php
// Database
define('DB_HOST', 'localhost');
define('DB_NAME', 'nikahin');
define('DB_USER', 'nikahin');
define('DB_PASS', 'yourpassword');

// App URL (no trailing slash)
define('APP_URL', 'https://nikahin.id');

// Claude API
define('CLAUDE_API_KEY', getenv('CLAUDE_API_KEY') ?: 'sk-ant-...');
define('CLAUDE_MODEL',   'claude-sonnet-4-6');

// Dev flags — TURN OFF FOR PRODUCTION
define('DEV_MODE_SHOW_OTP', true);   // shows OTP on screen
define('SKIP_PAYMENT',      true);   // auto-marks orders as paid
```

### 5. Set permissions

```bash
chown -R www-data:www-data /var/www/html/nikahin
chmod -R 755 /var/www/html/nikahin
chmod -R 775 /var/www/html/nikahin/uploads
```

### 6. Enable Apache modules

```bash
sudo a2enmod rewrite headers expires deflate
sudo systemctl restart apache2
```

Make sure `AllowOverride All` is set for the directory in your Apache vhost:

```apacheconf
<Directory /var/www/html/nikahin>
    AllowOverride All
    Require all granted
</Directory>
```

### 7. Test it

Open your browser to `http://yourserver/nikahin/`. You should see the landing page.

The seeded admin account is `admin@nikahin.local` — log in with that email; the OTP will appear on screen because `DEV_MODE_SHOW_OTP` is `true`.

## Configuration

All configuration lives in `config/config.php`. Key settings:

| Constant | Default | Purpose |
|---|---|---|
| `DEV_MODE_SHOW_OTP` | `true` | Show OTP code on screen instead of sending SMS. **Turn OFF in production.** |
| `SKIP_PAYMENT` | `true` | Auto-marks orders as PAID without payment gateway. **Turn OFF in production.** |
| `CLAUDE_API_KEY` | env var | Your Anthropic API key. If empty, AI tier falls back to a default preset. |
| `CLAUDE_MODEL` | `claude-sonnet-4-6` | Model used for design generation. Use `claude-opus-4-7` for highest quality. |
| `PRICE_BASIC` | `50000` | Price in IDR for Basic tier. |
| `PRICE_AI` | `150000` | Price in IDR for AI tier. |
| `MAX_UPLOAD_SIZE` | `5242880` | Max file upload size (5MB). |

## Development Mode

Out of the box, the app runs in **development mode** so you can test the full flow without external services:

- **OTP shown on screen** — When you register or log in, the OTP code is displayed in a yellow banner. No SMS gateway needed.
- **Payment skipped** — Clicking "Lanjut ke Pembayaran" auto-marks the order as PAID and moves to AI generation.
- **Without a Claude API key** — The AI tier falls back to a default design preset, so you can test the publish flow.

To test the full AI flow:

1. Set `CLAUDE_API_KEY` in `config/config.php` or as an environment variable.
2. Choose AI tier when creating an invitation.
3. Fill the form, click submit. Generation runs synchronously (~30s–2min).

## Going to Production

Before deploying to a live server:

### 1. Turn off dev flags

```php
define('DEV_MODE_SHOW_OTP', false);
define('SKIP_PAYMENT',      false);
```

### 2. Hook up a real OTP provider

Edit `includes/auth.php`, method `Auth::issueOtp()`. Currently it stores the code and (in dev mode) shows it; replace the SMS/email sending stub with your provider:

```php
// Example: Twilio
$twilio = new Client($accountSid, $authToken);
$twilio->messages->create($phone, [
    'from' => '+1234567890',
    'body' => "Kode OTP nikahin Anda: {$code}"
]);

// Example: Fonnte (Indonesian WhatsApp gateway)
$ch = curl_init('https://api.fonnte.com/send');
curl_setopt($ch, CURLOPT_POSTFIELDS, http_build_query([
    'target'  => $phone,
    'message' => "Kode OTP nikahin Anda: {$code}",
]));
curl_setopt($ch, CURLOPT_HTTPHEADER, ['Authorization: ' . FONNTE_TOKEN]);
curl_exec($ch);
```

### 3. Implement payment gateway

Create `invitation/payment.php` to handle Midtrans, Xendit, or your gateway of choice. After payment success webhook:

```php
DB::run("UPDATE orders SET status='paid', paid_at=NOW() WHERE id = ?", [$orderId]);
DB::run("UPDATE invitations SET status='paid' WHERE id = ?", [$invId]);
// Then redirect to invitation/generate.php
```

### 4. Move AI generation to a queue

For production traffic, replace the synchronous call in `invitation/generate.php` with a job queue (Redis + Beanstalkd, RabbitMQ, AWS SQS, etc.). The current implementation blocks the user's browser during generation, which is fine for MVP but won't scale.

### 5. Tighten the CSP header

Edit `.htaccess` to remove `'unsafe-inline'` from `script-src` after refactoring inline scripts into separate files.

### 6. Force HTTPS

Uncomment the HTTPS redirect block at the top of `.htaccess`.

### 7. Nginx alternative

If using Nginx instead of Apache:

```nginx
server {
    listen 443 ssl;
    server_name nikahin.id;
    root /var/www/nikahin;
    index index.php;

    location / {
        try_files $uri $uri/ @rewrite;
    }

    location @rewrite {
        rewrite ^/v/([a-zA-Z0-9-]+)/g/([a-zA-Z0-9]+)/?$ /view.php?slug=$1&token=$2 last;
        rewrite ^/v/([a-zA-Z0-9-]+)/?$                   /view.php?slug=$1          last;
    }

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.1-fpm.sock;
    }

    location ~ /(config|database|includes|BasicTemplate)/ {
        deny all;
    }

    location ~ /uploads/.*\.(php|phtml|pl|py|sh)$ {
        deny all;
    }
}
```

## API Endpoints

All POST endpoints require `_csrf` token. Auth is enforced via session except where noted.

| Method | Endpoint | Auth | Purpose |
|---|---|---|---|
| POST | `/api/save-draft.php` | User | Auto-save form draft |
| POST | `/api/upload.php` | User | Upload photo (multipart) |
| POST | `/api/upload.php?action=delete` | User | Remove photo |
| GET  | `/api/generate.php?id={inv_id}` | User | Poll generation status |
| POST | `/api/rsvp.php` | Public | Submit RSVP (slug + optional token) |
| POST | `/api/guestbook.php` | Public | Submit guestbook message |

## Database Schema

Eight core tables:

| Table | Purpose |
|---|---|
| `users` | Account records (email, phone, status, is_admin) |
| `otp_codes` | One-time codes (hashed, 5min TTL) |
| `invitations` | The product itself (tier, status, slug, profile_json, design_json) |
| `invitation_assets` | Uploaded files (groom_photo, bride_photo, prewedding, generated, etc.) |
| `orders` | Payment records (kept even when SKIP_PAYMENT for traceability) |
| `generations` | AI generation logs (tokens, model, errors) |
| `guests` | Guest list with personalized link tokens + RSVP state |
| `guestbook` | Guest messages (with hidden flag for moderation) |
| `gifts` | Gift transaction log (optional, for confirmed gifts) |
| `audit_log` | Admin actions audit trail |

See `database/schema.sql` for full DDL.

## Customization

### Change the design tokens

Edit CSS variables at the top of `css/style.css` and `css/app.css`:

```css
:root {
  --gold:        #C8943A;
  --beige:       #F4EDE0;
  --ecru:        #E8DCC4;
  --champagne:   #F0D9A8;
  --brown:       #5C3A1E;
  --dark:        #2C1A0E;
}
```

### Change pricing

Edit `config/config.php`:

```php
define('PRICE_BASIC', 75000);   // Was 50000
define('PRICE_AI',    250000);  // Was 150000
```

### Add a new template

Currently the `BasicTemplate/template.php` is the only renderer. To add variations:

1. Duplicate `BasicTemplate/` to e.g. `MinimalTemplate/`.
2. Add a `template_id` column to `invitations`.
3. In `view.php`, switch on `template_id` to include the right renderer.

### Change Claude's design output

Edit the system prompt in `includes/claude_api.php`, method `ClaudeApi::generateDesign()`. The prompt enforces:

- Color palette (5 hex colors)
- Typography (from a whitelisted list of Google Fonts)
- Indonesian copy (welcome, invocation matching religion, story)
- Section order
- RSVP prompt
